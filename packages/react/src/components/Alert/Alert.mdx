Alert displays a contextual message banner â€” informational, confirmatory, or
cautionary. Colors, borders, and foreground text are driven entirely by design
tokens, so they adapt to the active theme automatically.

## When to use

- **Use an Alert** for static, inline feedback that does not require user action
  (page-level notices, form submission results, system status).
- **Prefer `info`** for neutral guidance; use `success`, `warning`, or `danger`
  only when the semantic meaning genuinely matches â€” overusing `danger` trains
  users to ignore it.
- Reach for a toast or dialog instead when the message is transient or demands
  an explicit user response.

```tsx
import { Alert, AlertTitle, AlertDescription } from "@jasonruesch/react";
```

## Anatomy

Alert is a composite component made up of three sub-components:

| Sub-component | Element | Role |
|---|---|---|
| `Alert` | `<div>` | Outer container; applies the variant styles and the appropriate ARIA live role |
| `AlertTitle` | `<p>` | Bold heading that names the message category |
| `AlertDescription` | `<p>` | Supporting body text with full contrast against the tinted background |

## Accessibility

- `danger` and `warning` variants render `role="alert"` (assertive live
  region), so screen readers announce the content immediately without the user
  having to focus the element.
- `info` and `success` variants render `role="status"` (polite live region),
  which waits for the user to finish their current task before announcing.
- `AlertDescription` intentionally avoids opacity de-emphasis â€” dimming the
  semantic foreground color on a tinted variant background would drop contrast
  below the 4.5:1 WCAG AA threshold.
- Alert is not interactive; do not place it inside a button or link, and do not
  add `tabIndex` to the container element.
