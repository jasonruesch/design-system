A CSS grid layout primitive with token-based column count and gap sizing. Column
count and spacing are driven by design tokens, so layouts adapt consistently
across the active theme.

## When to use

- **Use Grid** when you need a multi-column layout with consistent gutters —
  card grids, form field rows, image galleries.
- **Use Flexbox utilities** instead when items need to wrap freely or align
  along a single axis without fixed column counts.
- Prefer lower column counts (`2`–`4`) for content grids; use `12` only when
  you need fine-grained column spanning control.

## Import

```tsx
import { Grid } from "@jasonruesch/react";
```

## Accessibility

- Renders a native `<div>` by default; pass `as` to swap the element (e.g.
  `as="ul"`) when the grid content is semantically a list.
- Grid itself carries no landmark role — wrap it in the appropriate semantic
  container (`<main>`, `<section>`, etc.) so screen readers understand its
  context.
- Child elements are responsible for their own focus order; Grid does not alter
  the DOM order, so reading order follows source order automatically.
